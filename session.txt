================================================================================
WHAT THIS IS
================================================================================
Clone-and-run Docker Compose local AI stack:
  Ollama -> LiteLLM -> Open WebUI + OpenClaw -> SearXNG, optional Traefik proxy.
Idempotent install.sh: detects GPU, picks proxy/TLS mode, renders per-host config
+ secrets, sets bind-mount perms, prompts Watchtower frequency, auto-updates.
Built to run on CLEAN hosts with no pre-existing infrastructure.

RESOLVED THIS SESSION (was the big blocker)
- OpenClaw Control UI would not load over plain HTTP from a remote browser
  (browser secure-context rule; allowInsecureAuth / dangerouslyDisableDeviceAuth
  are BOTH ineffective for remote LAN HTTP — confirmed against the OpenClaw bug
  tracker). Fix = provide a real HTTPS secure context, self-contained:
  -> Bundled Traefik with a self-signed cert (Traefik default TLS). No external
     DNS / domain / CA needed; works by raw IP. Each fronted service gets its own
     HTTPS entrypoint/port (no Host rules), so it's portable to any clean box.

KEY DESIGN DECISIONS
- Earlier homelab-specific fix (Caddy LXC + *.true-frontier.com wildcard +
  split-horizon DNS) was REJECTED — stack must not assume host infra. Traefik-
  in-stack with self-signed TLS is the portable replacement.
- Base compose publishes NO user-facing ports; overlays add them (Compose can't
  remove a port via overlay, so ports must be additive). Overlays:
    compose.direct-webui / direct-openclaw  (plain HTTP)
    compose.proxy-base                       (Traefik service)
    compose.front-webui / front-openclaw     (Traefik HTTPS routers + ports)
- PROXY_MODE = none | openclaw | all  (installer prompt, --proxy flag, or
  reconfigure-proxy). openclaw mode = HTTPS dashboard for OpenClaw, WebUI direct.
