WHAT THIS IS
================================================================================
Clone-and-run Docker Compose local AI stack, built for CLEAN hosts (no assumed infra):

secrets, sets bind-mount perms, wires SearXNG/Discord, auto-updates via Watchtower.

================================================================================
OPENCLAW TUNING — the hard-won correct keys (this version = 2026.6.11)
================================================================================
- Idle/timeout watchdog is PER-PROVIDER, not global:
    models.providers.litellm.timeoutSeconds = 300     (CORRECT)
    agents.defaults.llm.idleTimeoutSeconds            (WRONG — schema rejects
      "llm"; this exact key BOOT-LOOPED the gateway. Do not use. Old guides cite it.)
- tools.profile controls the agent tool loop. Allowed: minimal|coding|messaging|full.
  "coding" (the prior default) made a plain "hi" fire web_fetch etc → multi-call
  loop → timeouts on CPU. Set to "minimal" for CPU.
- Installer now auto-sets tools.profile from GPU: cpu/none -> minimal, gpu -> coding.
- Discord is an external PLUGIN: needs `openclaw plugins install @openclaw/discord`
  (installer's `discord` command now does this) + channels.discord.enabled +
  token {source:env,id:DISCORD_BOT_TOKEN} + plugins.allow:["discord"].
- SearXNG base URL inside the stack = http://searxng:8080 (pre-wired via
  SEARXNG_BASE_URL; NOT localhost/8888).
- Config edits: ALWAYS use `openclaw config set <path> <val>` — it validates
  before writing, so a bad key errors instead of boot-looping. Hand-editing the
  JSON is what caused the boot loops.
